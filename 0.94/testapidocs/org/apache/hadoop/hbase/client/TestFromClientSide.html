      • testPoolBehavior

        public void testPoolBehavior()
                      throws IOException,
                             InterruptedException
        This test demonstrates how we use ThreadPoolExecutor. It needs to show that we only use as many threads in the pool as we have region servers. To do this, instead of doing real requests, we use a SynchronousQueue where each put must wait for a take (and vice versa) so that way we have full control of the number of active threads.
        Throws:
        IOException
        InterruptedException
