      • testUpsertMSLAB

        public void testUpsertMSLAB()
                     throws Exception
        Test a pathological pattern that shows why we can't currently use the MSLAB for upsert workloads. This test inserts data in the following pattern: - row0001 through row1000 (fills up one 2M Chunk) - row0002 through row1001 (fills up another 2M chunk, leaves one reference to the first chunk - row0003 through row1002 (another chunk, another dangling reference) This causes OOME pretty quickly if we use MSLAB for upsert since each 2M chunk is held onto by a single reference.
        Throws:
        Exception
